Windows下搭建MapReduce开发环境

发表于 2017-10-10 | 分类于 大数据

简介

搭建好了Hadoop框架下的服务器集群后，我们需要一个master主机通过SSH控制我们的slaves服务器。我们可以通过SSH直接在命令行里写好Java文件，然后在yarn下运行，但这样步骤较多，不利于提高工作效率。所以在这里我们介绍用eclipse来直接对Hadoop进行上传、运行、本地调试，十分的方便快捷。

搭建步骤

  • eclipse for Java
  • hadoop-eclipse-plugin-2.7.0插件(根据Hadoop版本选择合适的插件)
  • Hadoop2.7.1环境(windows版)

安装eclipse插件

将hadoop-eclipse-plugin-2.7.0.jar拷贝至eclipse/plugins文件夹下，重启eclipse

部分版本eclipse不会自动更新插件列表，我们需要清空eclipse/configuration/org.eclipse.update文件夹下的配置文件，让eclipse重新搜索插件，重启eclipse就行。

  1. 进入eclipse，window->Preferences->Hadoop Map/Reduce，在这里配置Hadoop环境的路径

Hadoop部署简述

发表于 2017-10-10 | 分类于 大数据

Hadoop 简介

Hadoop是一个框架，用来在集群服务器上对大数据集进行分布式处理。Hadoop被设计成能够从单台服务器扩展到数以千计的服务器,其代码库自身就能在应用层侦测并处理硬件故障，因此能基于服务器集群提供高可用性的服务。

Hadoop 生态系统

  • HDFS: Hadoop分布式文件系统，是一种数据分布式保存机制，为其他很多工具提供了基础。

  • MapReduce: Hadoop的主要执行框架，一个分布式、并行处理的百年城模型，MapReduce能把任务分为map(映射)和reduce(化简)。由于MapReduce工作原理的特性，Hadoop能以并行的方式访问数据，从而实现快速访问数据。

关于hexo新版本配置问题

发表于 2017-10-09 | 分类于 杂项

hexo+GitHub配置的教程网上有很多，但是新版Hexo有几个地方会出错，在这里简单地总结一下

Deployer not found: github

在执行hexo deploy时，新版hexo会提示以上错误。这是因为新版部分配置文件要求进行了修改。下面是处理过程：

  1. 将_config.yml配置文件中的deploy下type字段后的github改为git

  2. 在Git Bash中执行npm install hexo-deployer-git --save(还有些版本中将save置后仍会出错，可以执行npm install --save hexo-deployer-git)

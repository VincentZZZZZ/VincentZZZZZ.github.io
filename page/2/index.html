MapReduce学习笔记

发表于 2017-11-01 | 分类于 大数据

MapReduce 是现今一个非常流行的分布式计算框架，它被设计用于并行计算海量数据。
当你向MapReduce框架提交一个任务时，它会首先把整个作业拆分成若干个Map任务，然后分配到不同的节点上去执行，每一个Map任务处理输入数据中的一部分，当Map任务完成后，它会生成一些中间文件，这些中间文件将会作为Reduce任务的输入数据。Reduce任务的主要目标就是把前面若干个Map的输出汇总到一起并输出。

忙忙碌碌

发表于 2017-10-18 | 分类于 胡思乱想

坐在图书馆咖啡厅，带着：一台还有67%电量的电脑，已经没电变砖的手机，作业还没写的罪恶感，还有社团一摊事

最近忙炸了，刚刚扔上那篇中文分词是上周做的，到这周才有一点时间来随便写写。

社团纳新真正自己接手过来发现的确是会忙得手忙脚乱，也许是策划没写好得原因。但是正如学长说的：参加活动和策划活动完全不是一回事。宣讲会上，埋了一个彩蛋，在没打招呼的情况下，给研究生的前前前任主席打电话。但是学长说的故事还是令我印象深刻：“我刚来大学时候，五六年前了，我的学长带着我逛了逛校园，在食堂吃了个饭。当时我的学长就和我说了一句话：这里的食堂 很便宜，但是时间 很贵。”

的确啊，大学一年过去了，发生了太多太多的事，多到我已经记不清我刚来大学时候的心态，记不得曾经的我也坐在下面听着学姐学长们分享着经验。真正轮到我们的时候，我们甚至都没做好准备。

但是哪有那么多次机会给人准备，事情接踵而来，时间很贵，回不来的。

我的确需要个计划。过去的一年我学到了很多，但是什么也没做。因为我没有目标，我没有规划。直到最近社团纳新，才知道计划不全多么麻烦。能化解一次，未必能化解第二次问题。

我貌似不知道自己想要什么，只知道把成绩弄上去，技术多学点，这的确是个问题，也许是时候抽个时间想想这件事了。

Hadoop部署简述

发表于 2017-10-10 | 分类于 大数据

Hadoop 简介

Hadoop是一个框架，用来在集群服务器上对大数据集进行分布式处理。Hadoop被设计成能够从单台服务器扩展到数以千计的服务器,其代码库自身就能在应用层侦测并处理硬件故障，因此能基于服务器集群提供高可用性的服务。

Hadoop 生态系统

  • HDFS: Hadoop分布式文件系统，是一种数据分布式保存机制，为其他很多工具提供了基础。

  • MapReduce: Hadoop的主要执行框架，一个分布式、并行处理的百年城模型，MapReduce能把任务分为map(映射)和reduce(化简)。由于MapReduce工作原理的特性，Hadoop能以并行的方式访问数据，从而实现快速访问数据。

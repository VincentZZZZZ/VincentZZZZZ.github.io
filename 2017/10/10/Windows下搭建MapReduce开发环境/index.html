Windows下搭建MapReduce开发环境

发表于 2017-10-10 | 分类于 大数据

简介

搭建好了Hadoop框架下的服务器集群后，我们需要一个master主机通过SSH控制我们的slaves服务器。我们可以通过SSH直接在命令行里写好Java文件，然后在yarn下运行，但这样步骤较多，不利于提高工作效率。所以在这里我们介绍用eclipse来直接对Hadoop进行上传、运行、本地调试，十分的方便快捷。

搭建步骤

  • eclipse for Java
  • hadoop-eclipse-plugin-2.7.0插件(根据Hadoop版本选择合适的插件)
  • Hadoop2.7.1环境(windows版)

创建Maven项目

New->other->Maven

添加Maven依赖

打开网页www.mvnrepository.com，搜索Hadoop，分别打开Apache Hadoop Common、Apache Hadoop HDFS、Apache Hadoop Client Common、Apache Hadoop Core，分别复制代码，加入pom.xml文件中。

安装eclipse插件

将hadoop-eclipse-plugin-2.7.0.jar拷贝至eclipse/plugins文件夹下，重启eclipse

部分版本eclipse不会自动更新插件列表，我们需要清空eclipse/configuration/org.eclipse.update文件夹下的配置文件，让eclipse重新搜索插件，重启eclipse就行。

配置eclipse插件

  1. 进入eclipse，window->Preferences->Hadoop Map/Reduce，在这里配置Hadoop环境的路径

  2. window->Show View->Other，点击Map/Reduce Locations打开插件，点击新出现窗口右上角的new，弹出插件配置窗口，更改Location name，将Host和Port改为设置Hadoop框架时的Host和Port，完成

如果是主机名，需要配置一下和IP的对应关系

替换Hadoop环境文件

因为是在eclipse运行，所以需要替换Hadoop环境文件，windows下的文件需要重新编译一下，在网上可以找到下载。

对Hadoop进行操作

在配置完成后，上传等操作就可以直接右击资源栏里的Hadoop插件，就可以上传到服务器集群的Hadoop里，然后直接能在本地运行MapReduce.
